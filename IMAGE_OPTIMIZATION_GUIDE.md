# دليل ضغط وتحسين الصور

## الطريقة الأولى: ضغط جميع الصور مرة واحدة

### الخطوات:

1. تثبيت الحزمة المطلوبة:
```bash
npm install
```

2. تشغيل السكريبت الكامل (ضغط + تحديث المسارات):
```bash
npm run optimize-all
```

أو خطوة بخطوة:

```bash
# الخطوة 1: ضغط وتحويل الصور
npm run optimize-images

# الخطوة 2: تحديث المسارات في الكود
npm run update-image-paths
```

### ماذا يفعل السكريبت؟
- يبحث عن جميع الصور في مجلد `public`
- يحول كل الصور إلى WebP (أصغر حجماً بنفس الجودة)
- يضغط الصور بجودة 85% (توازن ممتاز بين الحجم والجودة)
- يحذف الملفات القديمة تلقائياً
- يعرض لك كم وفرت من المساحة

### ملاحظات مهمة:
- ⚠️ السكريبت يحذف الصور القديمة! اعمل نسخة احتياطية أولاً
- الصور ستتحول من `.jpg`, `.png`, `.jpeg` إلى `.webp`
- ستحتاج لتحديث مسارات الصور في الكود

---

## الطريقة الثانية: استخدام Next.js Image Component

### المميزات:
- ضغط تلقائي عند الطلب
- تحويل تلقائي لـ WebP
- Lazy loading تلقائي
- لا حاجة لتغيير أسماء الملفات

### مثال:
```tsx
import Image from 'next/image';

// بدلاً من:
<img src="/image.jpg" alt="صورة" />

// استخدم:
<Image 
  src="/image.jpg" 
  alt="صورة"
  width={800}
  height={600}
  quality={85}
/>
```

---

## أيهما أفضل؟

### استخدم السكريبت (الطريقة الأولى) إذا:
- عندك صور كثيرة جداً (400+ صورة)
- تريد تقليل حجم المشروع بشكل دائم
- الصور ثابتة ولا تتغير كثيراً

### استخدم Next.js Image (الطريقة الثانية) إذا:
- تريد حل سريع بدون تعديل الملفات
- الصور تتغير باستمرار
- تريد lazy loading تلقائي
